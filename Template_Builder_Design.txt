# End-to-end design for a lightweight, placeholder-driven, template builder and renderer

This design integrates the agreed enhancements: global placeholder registry, composable templates, multi-datasource abstraction layer, preview with sample data, pluggable renderers, marketplace/sharing, versioning & audit, and a lightweight frontend. Points 9 and 10 are intentionally excluded.

---

## Product vision and core principles

- **Simplicity first:** Minimal UI, block-based editor, predictable behavior, easy onboarding.
- **Reusability:** Global placeholder registry and reusable blocks/sections across templates.
- **Extensibility:** Pluggable renderers and datasource connectors; composable structure over monoliths.
- **Observability and governance:** Versioning, diffs, audit logs, reproducible renders.
- **Performance-minded:** Batch datasource calls, caching sample data for previews, asynchronous rendering for large jobs.

---

## High-level architecture

- **Frontend (Builder + Preview):**
  - Drag-and-drop canvas for blocks (Text, Table, Image, Section).
  - Placeholder palette sourced from the global registry or template-scoped definitions.
  - Preview using sample values for instant feedback.
  - Outputs chosen at template level (HTML, DOCX, PDF, XLSX, Markdown).
- **Backend Service (Dockerized):**
  - REST API for templates, blocks, placeholders, datasources, registry, marketplace, and rendering jobs.
  - Resolver engine: resolves placeholders (generate SQL, execute, format).
  - Multi-datasource abstraction layer: fetch and merge datasets into a unified context.
  - Renderer plugins: output-specific renderers (HTML, DOCX, PDF via HTML, XLSX, Markdown).
  - Versioning and audit: immutable versions, diffs, job logs.
- **Postgres:**
  - Storage for templates, blocks, placeholders, datasources, registry entries, marketplace items, versions, jobs, audit logs.
- **Optional:** Redis (queue + cache) for previews and renders; object storage for generated documents.

## Data model (Postgres core schema)

### Templates and versions

```sql
CREATE TABLE templates (
  template_id        UUID PRIMARY KEY,
  name               TEXT NOT NULL,
  description        TEXT,
  status             TEXT NOT NULL CHECK (status IN ('draft','published','archived')),
  output_target      TEXT NOT NULL CHECK (output_target IN ('html','docx','pdf','xlsx','md')),
  root_layout_json   JSONB NOT NULL,   -- structure composed from reusable blocks/sections
  created_by         TEXT NOT NULL,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE template_versions (
  version_id         UUID PRIMARY KEY,
  template_id        UUID NOT NULL REFERENCES templates(template_id) ON DELETE CASCADE,
  version_number     INT NOT NULL,
  layout_json        JSONB NOT NULL,   -- fully resolved snapshot
  output_target      TEXT NOT NULL,
  change_summary     TEXT,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (template_id, version_number)
);
```

### Global placeholder registry and template bindings

```sql
CREATE TABLE placeholders_registry (
  registry_id        UUID PRIMARY KEY,
  name               TEXT NOT NULL,            -- e.g., loan_number
  category           TEXT,                     -- e.g., loan, crm
  prompt             TEXT NOT NULL,            -- SQL gen prompt (LLM/rules)
  format_json        JSONB,                    -- formatting spec
  datasource_id      UUID NOT NULL REFERENCES datasources(datasource_id),
  sample_value       TEXT,
  metadata           JSONB,                    -- type, constraints, preview hints
  is_active          BOOLEAN NOT NULL DEFAULT TRUE,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE template_placeholders (
  template_placeholder_id UUID PRIMARY KEY,
  template_id        UUID NOT NULL REFERENCES templates(template_id) ON DELETE CASCADE,
  registry_id        UUID REFERENCES placeholders_registry(registry_id),
  name               TEXT NOT NULL,
  override_prompt    TEXT,                     -- optional per-template
  override_format    JSONB,                    -- optional per-template
  override_datasource_id UUID,                 -- optional per-template
  sample_value       TEXT,
  metadata           JSONB,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (template_id, name)
);
```

### Datasources and abstraction

```sql
CREATE TABLE datasources (
  datasource_id      UUID PRIMARY KEY,
  name               TEXT NOT NULL UNIQUE,
  type               TEXT NOT NULL CHECK (type IN ('snowflake','postgres','mysql','rest','graphql')),
  config_json        JSONB NOT NULL,           -- DSN/creds/schema/base URL
  is_active          BOOLEAN NOT NULL DEFAULT TRUE,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Optional logical models that unify multiple sources via backend composition
CREATE TABLE logical_models (
  model_id           UUID PRIMARY KEY,
  name               TEXT NOT NULL UNIQUE,
  definition_json    JSONB NOT NULL,           -- mapping/joins, field aliases
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### Blocks and composable sections

```sql
CREATE TABLE blocks_library (
  block_id           UUID PRIMARY KEY,
  name               TEXT NOT NULL,            -- e.g., "Loan header"
  type               TEXT NOT NULL CHECK (type IN ('text','table','image','section')),
  block_json         JSONB NOT NULL,           -- content with tokens and sub-blocks
  tags               TEXT[],
  created_by         TEXT NOT NULL,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE template_blocks (
  template_block_id  UUID PRIMARY KEY,
  template_id        UUID NOT NULL REFERENCES templates(template_id) ON DELETE CASCADE,
  block_id           UUID REFERENCES blocks_library(block_id),
  position_index     INT NOT NULL,             -- order in layout
  override_json      JSONB,                    -- local overrides or bindings
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### Marketplace and sharing

```sql
CREATE TABLE marketplace_items (
  item_id            UUID PRIMARY KEY,
  type               TEXT NOT NULL CHECK (type IN ('template','block','placeholder')),
  source_id          UUID NOT NULL,            -- points to templates/blocks/registry
  name               TEXT NOT NULL,
  description        TEXT,
  owner              TEXT NOT NULL,
  license            TEXT DEFAULT 'Community',
  rating             NUMERIC(2,1),
  downloads          INT DEFAULT 0,
  tags               TEXT[],
  is_public          BOOLEAN NOT NULL DEFAULT TRUE,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### Rendering jobs and audit

```sql
CREATE TABLE render_jobs (
  job_id             UUID PRIMARY KEY,
  template_id        UUID NOT NULL REFERENCES templates(template_id),
  version_number     INT,                      -- if rendering a specific version
  output_target      TEXT NOT NULL,
  status             TEXT NOT NULL CHECK (status IN ('queued','running','success','error')),
  runtime_params     JSONB,                    -- user-provided values if any
  result_location    TEXT,                     -- path/URL to artifact
  logs               TEXT,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE audits (
  audit_id           UUID PRIMARY KEY,
  entity_type        TEXT NOT NULL,            -- 'template','placeholder','render_job'
  entity_id          UUID NOT NULL,
  action             TEXT NOT NULL,
  actor              TEXT NOT NULL,
  summary            TEXT,
  details_json       JSONB,
  ts                 TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## Frontend design

### Core UI

- **Template selector:** Search, filter by status/tags; open existing or create new.
- **Placeholder palette:**
  - **Source toggle:** Global registry or template-specific.
  - **Display:** Name, category tag, datasource tag, sample value (hover tooltip).
- **Canvas blocks:**
  - **Text block:** ContentEditable with inline tokens `{{placeholder_name}}`.
  - **Table block:** Column headers and bindings; supports repeat over dataset.
  - **Image block:** URL or data-bound image placeholder.
  - **Section block:** Composable group of sub-blocks; import from blocks_library.
- **Inspector panel:**
  - **Block properties:** Alignment, font size, bindings, repeat configs.
  - **Token validation:** Warnings for unknown tokens; show sample preview.
- **Output target selector:** html/docx/pdf/xlsx/md at template level.
- **Preview:**
  - **HTML live preview:** Substitute sample values for quick validation.
  - **Table preview:** Show a few sample rows from sample datasets or static mock.
- **Version controls:** Save, publish, version notes, diff view.

### Interactions

- **Drag and drop tokens to text:** Inserts `{{token}}` at caret; context-aware whitespace.
- **Drag to table columns:** Assigns column binding; visual badge shows bound token.
- **Block reorder:** Drag sections/blocks; position_index updates.
- **Section reuse:** Import from blocks_library; optionally override content.

## Backend service API

### Templates

- **GET /templates**
  - **Label:** List
  - **Returns:** Minimal metadata and status.
- **GET /templates/{id}**
  - **Label:** Fetch
  - **Returns:** Root layout, bound placeholders, blocks.
- **POST /templates**
  - **Label:** Create
  - **Body:** name, output_target, root_layout_json.
- **PUT /templates/{id}**
  - **Label:** Update
  - **Body:** root_layout_json, output_target, status.
- **POST /templates/{id}/publish**
  - **Label:** Publish
  - **Body:** change_summary
  - **Effect:** Create immutable version snapshot.

### Placeholder registry and template bindings

- **GET /registry/placeholders**
  - **Label:** List
  - **Query:** category, active, text search.
- **POST /registry/placeholders**
  - **Label:** Create global placeholder
  - **Body:** name, prompt, format_json, datasource_id, metadata.
- **GET /templates/{id}/placeholders**
  - **Label:** List bound placeholders
- **POST /templates/{id}/placeholders**
  - **Label:** Bind placeholder
  - **Body:** registry_id or full definition (name, prompt, format_json, datasource_id).
- **PUT /templates/{id}/placeholders/{tpl_ph_id}**
  - **Label:** Override prompt/format/datasource
- **DELETE /templates/{id}/placeholders/{tpl_ph_id}**
  - **Label:** Remove binding

### Blocks and composable sections

- **GET /blocks**
  - **Label:** List reusable blocks
- **POST /blocks**
  - **Label:** Create block
  - **Body:** name, type, block_json, tags
- **POST /templates/{id}/blocks**
  - **Label:** Attach block to template with position_index
- **PUT /templates/{id}/blocks/{tpl_block_id}**
  - **Label:** Update overrides or position_index
- **DELETE /templates/{id}/blocks/{tpl_block_id}**
  - **Label:** Remove block from template

### Datasources and logical models

- **GET /datasources**
  - **Label:** List
- **POST /datasources**
  - **Label:** Register datasource
  - **Body:** name, type, config_json
- **GET /logical-models**
  - **Label:** List
- **POST /logical-models**
  - **Label:** Create
  - **Body:** definition_json (field maps, source composition)

### Rendering and preview

- **POST /preview**
  - **Label:** Preview with sample values
  - **Body:** template_id, optional overrides for sample values
  - **Returns:** HTML preview payload (no datasource calls)
- **POST /render**
  - **Label:** Render full document
  - **Body:** template_id, output_target (optional override), runtime_params
  - **Returns:** job_id
- **GET /render/{job_id}**
  - **Label:** Poll status and result

### Marketplace

- **GET /marketplace**
  - **Label:** List public items
- **POST /marketplace**
  - **Label:** Publish item (template, block, placeholder)
- **POST /marketplace/import**
  - **Label:** Import into account

## Resolver engine and multi-datasource abstraction

### Resolution strategy

- **Collect tokens used in layout:** Extract unique placeholders present across blocks.
- **Map placeholders to registry/template bindings:** Build a resolution plan.
- **Group by datasource:** Batch SQL generation and execution per datasource to minimize round trips.
- **Generate SQL for each group:**
  - **Input:** prompt, runtime_params, optional logical model hints.
  - **Output:** parameterized SQL + params.
- **Execute queries:** Use connector for datasource type; return structured results (scalars, rows).
- **Apply formatting:** Use format_json per placeholder:
  - **Label:** Date, currency, numeric precision, string transforms.
- **Build context:** A flat map for scalar tokens and datasets for tables:
  - **Label:** context.values[name] for text bindings; context.datasets[source] for table repeats.

### Logical models (optional but recommended)

- **Definition:** A JSON schema describing how multiple datasources compose into a unified model with aliased fields.
- **Use:** Placeholders can refer to logical fields; backend translates to appropriate datasource queries and merges.
- **Benefit:** Decouples templates from physical schemas; easier evolution.

## Renderer plugins

- **HTML renderer:**
  - **Input:** layout_json, context.values, context.datasets.
  - **Process:** Token replacement in text blocks; table repeat expansion in HTML.
  - **Output:** Final HTML string.
- **DOCX renderer:**
  - **Input:** Same context; map blocks to paragraphs/tables.
  - **Process:** Use a lightweight templater (e.g., python-docx) with token replacement and table population.
  - **Output:** DOCX file path/URL.
- **PDF renderer:**
  - **Input:** HTML output from HTML renderer.
  - **Process:** Headless conversion to PDF (e.g., wkhtmltopdf or Chromium).
  - **Output:** PDF path/URL.
- **XLSX renderer:**
  - **Input:** Table blocks and datasets.
  - **Process:** Use xlsxwriter/openpyxl to create sheets; populate columns with bindings.
  - **Output:** XLSX path/URL.
- **Markdown renderer:**
  - **Input:** Blocks mapped to markdown; tables using markdown table syntax.
  - **Process:** Token replacement and repeat expansion.
  - **Output:** MD file or string.

### Renderer contract

- **Request:**
  - **Label:** layout_json, context, output_target, options (fonts, page size).
- **Response:**
  - **Label:** artifact location, metadata (pages, size).

## Versioning, diffs, and audit

- **Versioning flow:**
  - **Label:** Draft → Publish → Freeze snapshot in template_versions.
- **Diffs:**
  - **Label:** Compare layout_json between version N and N+1; show changes in blocks, bindings, tokens.
- **Audit:**
  - **Label:** Record actions (create/update/publish/render), actor, summary, details_json for reproducibility.

## Performance and scalability

- **Batching:** Resolve placeholders per datasource in batches; reuse connections.
- **Caching previews:** Cache sample-substituted previews keyed by template_id + version + small hash of layout.
- **Async rendering:** Large jobs queued and processed by worker pods; frontend polls job status.
- **Resource management:** Renderer workers limit concurrency; PDF renderer uses a shared headless browser pool.

## Lightweight frontend implementation notes

- **Stack:** React + DnD Kit; Zustand/Context for state; no heavy WYSIWYG.
- **Blocks:** Minimal CSS; contenteditable for text; simple table editor for columns.
- **Tokens:** Use `{{token}}`; prevent invalid characters; autocomplete on “{{”.
- **Accessibility:** Keyboard insertion and search in palette; hints for sample value preview.
- **Error surface:** Red underline for unknown tokens, yellow for missing sample values.

## Example end-to-end workflow

1. **Registry author defines placeholders:**
   - **Label:** “loan_number”, prompt for SQL, format_json, datasource “loan_core”.
2. **Designer builds template:**
   - **Label:** Select placeholders from palette; drag into text and table blocks; choose output “pdf”.
3. **Publish template:**
   - **Label:** Version 3 created with snapshot; marketplace publish optional.
4. **User requests render:**
   - **Label:** POST /render with runtime_params `{ "loan_number": "12345" }`.
5. **Backend resolves:**
   - **Label:** Group queries by datasource; execute; format values.
6. **Render PDF:**
   - **Label:** HTML renderer → PDF converter; save to object storage; update job status.
7. **Download:**
   - **Label:** GET /render/{job_id} returns success + link.

## Risks and mitigations

- **Misaligned tokens:** Mitigate with palette-driven insertion and token validation.
- **Datasource drift:** Logical models help decouple templates; versioned registry entries.
- **Long render times:** Async jobs, batching, PDF conversion pooling.
- **Template sprawl:** Blocks_library and marketplace promote reuse; versioning keeps order.

## 4. Resolver Engine Pseudocode

```python
def resolve_template(template_id, runtime_params):
    template = db.get_template(template_id)
    placeholders = db.get_placeholders(template_id)

    context = {"values": {}, "datasets": {}}

    # Group placeholders by datasource
    grouped = group_by_datasource(placeholders)

    for ds_id, ph_list in grouped.items():
        conn = connect_datasource(ds_id)
        for ph in ph_list:
            sql, params = generate_sql(ph.prompt, runtime_params)
            rows = conn.execute(sql, params)
            formatted = apply_format(rows, ph.format_json)
            if is_scalar(formatted):
                context["values"][ph.name] = formatted
            else:
                context["datasets"][ph.name] = formatted

    return context
```
